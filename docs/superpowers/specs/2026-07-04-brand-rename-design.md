## 二、侧边栏 SVG Logo 设计

**设计方向：音符 + 书页组合**

- **结构**：方形圆角底框（36×36，rx=10），垂直分割
- **左半**：三条平行横线（表示日记本的横线格）+ 一条稍粗竖线（书脊），用 `<path>` 或 `<line>` 绘制，颜色 `#0B0E0C`（深色）
- **右半**：音符图形（与现有音符相似的形态：竖线 + 符尾），颜色 `#0B0E0C`
- **背景渐变**：保持当前的线性渐变 `#4DB88D → #319F6A`

这样左右组合清晰传达「听歌（音符）+ 日记（横线格）」的双重含义，视觉上也与当前风格统一。

## 三、Favicon 设计

使用与侧栏 logo 相同的 SVG 图形，但去掉书页横线的细线细节（在小尺寸下不清晰），仅保留：
- 背景色块（渐变）
- 音符线条（简化，只保留关键笔触）
- 书页的竖线（书脊）

## 四、不变部分

- `localStorage` key 名称（`music_player_session` 等）→ 保持向后兼容
- `BroadcastChannel` 名称（`music_player_lyrics`）→ 保持歌词功能
- DB 表名、列名、API 路由 → 无需改动
- `js/ui.js:1200` 中的 `听歌笔记` — 这是「功能名（文章系统）」而非「品牌名」，保持不动
- `js/ui.js:1274` 的 `听歌笔记` 同理 — 是侧边栏导航项的「听歌笔记」功能名，不修改
