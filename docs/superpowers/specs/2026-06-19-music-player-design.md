# 音乐播放器 — 设计规格

## 背景
做一个基于 B站视频音频流的纯音频音乐播放器网页。起步歌曲：离别开出花、小幸运、匆匆那年、虞兮叹。支持循环/随机/单曲循环模式、收藏、自定义歌单。后续会扩展更多歌曲。

## 技术栈
- **前端**: 单 HTML + CSS + 原生 JS（无框架），localStorage 存储歌单/收藏
- **后端**: Python FastAPI，代理 B站 DASH 音频流，实时获取最新 URL 并流式转发
- **部署**: uvicorn 运行后端，浏览器直接打开 index.html

## 架构

```
浏览器 (index.html) <--audio stream--> FastAPI: /api/stream/<song_id>
                          ↑
                     B站 API (获取 DASH 音频 URL)
```

## 数据流
1. 前端请求 `/api/stream/1`（歌曲 ID）
2. 后端调用 B站视频信息 API 获取 DASH 音频流 URL
3. 后端用 httpx 流式请求音频，边下载边转发给前端
4. 前端 `<audio>` 标签播放，无需等待全部下载

## 功能规格

### 基础播放
- 歌曲列表显示歌曲名、序号、收藏状态
- 点击歌曲播放，底部播放栏显示当前歌曲名
- 播放/暂停、上一首、下一首按钮
- 进度条可拖拽跳转

### 播放模式（三态切换）
- 列表循环（默认）：播完最后一首回到第一首
- 单曲循环：当前歌曲无限循环
- 随机播放：随机选下一首

### 收藏系统 (localStorage)
- 空心/实心 ⭐ 切换收藏状态
- 收藏列表独立展示
- 收藏状态在歌曲列表中显示

### 自定义歌单 (localStorage)
- 新建歌单（输入名称）
- 从歌曲库添加歌曲到歌单
- 删除歌单中歌曲
- 删除整个歌单

### B站音频代理
- `/api/stream/<song_id>` — 获取 DASH 音频流并转发
- `/api/songs` — 返回歌曲列表（方便后续动态扩展）
- 支持 bv号 + 分P号 的视频定位

## 设计风格
- 青春清新：暖蜜桃粉 (#F5E6D3) + 柔白 (#FFFDF9) 底色
- 强调色：柔珊瑚 (#E8917B)
- 播放栏：底部固定，毛玻璃效果 (backdrop-filter: blur)
- 歌曲卡片：白底圆角、微阴影、hover 浮起
- 字体：系统无衬线字体，中文优化
